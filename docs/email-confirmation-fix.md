# Исправление проблемы с подтверждением Email

## Проблема
При регистрации пользователь получает письмо со ссылкой для подтверждения, но не может перейти по ссылке, так как redirect URL не настроен правильно.

## Решение

### 1. Настройка Supabase Redirect URLs

1. Откройте ваш проект в [Supabase Dashboard](https://supabase.com/dashboard)
2. Перейдите в **Authentication** → **Settings**
3. В разделе **Site URL** установите:
   ```
   http://localhost:3000
   ```

4. В разделе **Redirect URLs** добавьте следующие URL:
   ```
   http://localhost:3000/auth/callback
   http://localhost:3000/**
   ```
   
5. Нажмите **Save**

### 2. Настройка Email Templates (опционально)

1. В том же разделе **Authentication** → **Settings**
2. Прокрутите до **Email Templates**
3. Настройте шаблон **Confirm signup**:
   - Subject: `Подтвердите ваш email в TaskFlow`
   - Body: Используйте переменную `{{ .ConfirmationURL }}`

### 3. Включение подтверждения email

Убедитесь, что включена опция **"Enable email confirmations"**

### 4. Тестирование

1. Зарегистрируйте нового пользователя в приложении
2. Проверьте почту (включая папку "Спам")
3. Перейдите по ссылке в письме
4. Попробуйте войти в приложение

## Альтернативное решение (если не работает)

Если redirect URLs не работают, можно временно отключить подтверждение email:

1. В **Authentication** → **Settings**
2. Отключите **"Enable email confirmations"**
3. Пользователи смогут входить сразу после регистрации

## Для продакшена

Для продакшена настройте:
- Правильный домен вместо `localhost:3000`
- Настройте SMTP для отправки email
- Настройте SSL сертификат

## Как работает в приложении

1. **Регистрация**: Пользователь вводит email, пароль и имя
2. **Отправка письма**: Supabase отправляет письмо подтверждения
3. **Экран ожидания**: Приложение показывает экран с информацией о необходимости подтвердить email
4. **Подтверждение**: Пользователь переходит по ссылке в письме
5. **Вход**: После подтверждения пользователь может войти в систему
